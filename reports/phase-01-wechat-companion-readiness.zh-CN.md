# Phase 01 中文报告：微信陪伴通道配置就绪

## 1. 阶段目标

本阶段目标不是直接把微信真实接入上线，而是先把 Hermes 的微信陪伴通道做成安全、可配置、可观测的预备状态。

主人想要的是“在微信里像人一样聊天”。本阶段把这句话拆成工程边界：

- 可以自然、温柔、有上下文地聊天。
- 可以在主人允许后做低频主动问候。
- 可以把重要信息转成记忆候选。
- 不能伪装成真人。
- 不能绕过微信平台规则。
- 不能从微信直接执行公司、金钱、法务、HR、账号、交易等高风险动作。

## 2. 已完成内容

- 新增 `docs/WECHAT_COMPANION.md`，定义微信陪伴通道方案。
- 新增微信运行时配置项，默认全部安全关闭。
- 新增公众号和企业微信客服的密钥占位字段，只允许放在服务器 `.env`。
- `/health` 新增 `wechat` 状态块，只显示是否配置，不显示真实密钥。
- 明确微信优先使用官方通道：公众号或企业微信客服。
- 明确微信个人号桥接默认禁用，需要单独风险接受后才可讨论。
- 明确飞书 CLI/MCP 的定位：它们是受控工具层，不是第一入口层。
- 更新 README、架构、路线图、风险边界和 Phase 1 运行说明。
- 修复 runtime 在 systemd 停止时可能卡住的问题：停止信号会通过后台线程触发 shutdown，并在退出时关闭 server。
- 更新单元测试，覆盖微信配置加载和健康检查脱敏输出。

## 3. 当前效果

当前 Hermes 可以识别以下微信配置：

```text
HERMES_WECHAT_MODE=disabled
HERMES_WECHAT_CHANNEL=official_account
HERMES_WECHAT_PERSONA_MODE=companion
HERMES_WECHAT_PROACTIVE_CHAT=false
HERMES_WECHAT_MAX_DAILY_PROACTIVE_MESSAGES=3
HERMES_WECHAT_PERSONAL_BRIDGE_ENABLED=false
```

健康检查会返回类似状态：

```json
{
  "wechat": {
    "mode": "disabled",
    "channel": "official_account",
    "persona_mode": "companion",
    "proactive_chat": false,
    "personal_bridge_enabled": false
  }
}
```

它现在还没有启用真实微信回调，也没有发送微信消息。当前只是把安全配置、文档边界、健康检查和测试准备好。

## 4. 架构或决策变化

微信被正式定义为个人陪伴平面：

```text
微信官方通道
  -> Hermes 个人陪伴策略
  -> 模型回复
  -> 记忆候选
  -> Obsidian 审核入口
```

飞书继续作为公司管理平面：

```text
飞书 App / Channel SDK
  -> Hermes 公司工作流
  -> 飞书 CLI 或 MCP 工具层
  -> 审计日志
  -> 飞书 / Obsidian 总结
```

这次也明确了飞书 CLI/MCP 的答案：需要，但不是第一入口。第一入口应是飞书应用回调或 Channel SDK，CLI/MCP 适合作为受控执行工具。

## 5. 风险与边界

本阶段保持以下安全边界：

- 不提交任何真实微信密钥。
- 不提交任何服务器密码。
- 不启用微信个人号自动化。
- 不启用微信主动聊天。
- 不启用微信真实回调。
- 不允许微信直接执行高风险命令。
- 不允许从微信进行交易或资金动作。
- 不允许伪装成真人。

微信“像人一样”的可接受定义是：

- 语气自然。
- 回复有上下文。
- 记得经过治理的重要信息。
- 不连续刷屏。
- 能在主人批准后低频主动关心。

不可接受定义是：

- 伪装真人身份。
- 绕平台风控。
- 批量群发骚扰。
- 通过弱认证聊天入口执行公司或资金动作。

## 6. 需要主人确认的事项

进入真实微信接入前，需要主人确认：

- 使用公众号还是企业微信客服作为第一通道。
- 是否已有可配置的服务器公网域名和 HTTPS。
- 微信回调是否只给主人使用，还是也给员工/客户使用。
- 是否允许低频主动问候；如果允许，每天最多几条。
- 哪些内容可以进入 Obsidian 记忆候选，哪些永远不入库。
- 是否明确接受个人号桥接风险；默认不建议开启。

## 7. 下一阶段计划

推荐下一步仍然优先推进：

```text
Phase 2：飞书公司管理 MVP
```

如果主人想先看到微信聊天效果，可以开一个小阶段：

```text
Phase 1.6：微信官方通道回调 MVP
```

Phase 1.6 只做：

- 选择公众号或企业微信客服。
- 配置公网 HTTPS 回调。
- 校验 token 或签名。
- 接收主人一条消息。
- 返回一条简短中文陪伴回复。
- 写一条记忆候选到 Obsidian inbox。
- 保持个人号桥接禁用。

## 8. 记忆整理与清理结果

应进入长期记忆的内容：

- Hermes 微信通道当前是配置就绪，不是真实上线。
- 微信优先走官方通道，个人号桥接默认禁用。
- 微信用于个人陪伴和轻量提醒，不用于公司高风险动作。
- 飞书 CLI/MCP 是受控工具层，不是第一入口层。
- 主动聊天默认关闭，需要主人批准后按日上限启用。

不进入长期记忆的内容：

- 单次测试输出。
- 临时环境变量。
- 本地端口。
- 任何真实密钥、token、密码或回调验证值。

## 9. 风险复核结果

已复核：

- 没有提交真实微信密钥。
- 没有提交 AI API key。
- 没有提交服务器密码。
- 没有启用个人号桥接。
- 没有启用真实微信回调。
- 没有启用主动聊天。
- 没有新增交易、付款、HR、法务或外部承诺能力。

结论：

本阶段完成“微信陪伴通道配置就绪”。Hermes 现在可以安全表达微信接入状态，但还不会真的连微信聊天。下一步需要主人提供官方通道选择、域名/HTTPS 和对应平台凭据后，才能进入真实消息收发 MVP。
